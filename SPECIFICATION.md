# 项目需求说明书

## 一、项目概述

本项目以 C\+\+ 实现核心业务与通信逻辑，Qt 框架实现图形交互界面，完成两个独立进程（程序 A、程序 B）的网络双向通信：程序 A 通过 HTTP 协议主动调用程序 B 的指定函数并传递参数；程序 B 执行对应逻辑后，通过 HTTP 或 WebSocket 协议将带参数的结果回传给程序 A。

## 二、核心功能需求

### 1\. 整体通信架构

- 产出两套独立可执行程序：**程序 A（调用发起端）**、**程序 B（功能执行端）**，二者基于网络协议实现跨进程交互。

- 双向通信链路：

    - 正向调用链路：程序 A → HTTP 协议 → 程序 B，请求携带目标函数标识与入参参数

    - 反向回传链路：程序 B → HTTP / WebSocket 协议 → 程序 A，响应携带执行结果与返回参数

### 2\. 程序 A（调用端）功能

- 内置 HTTP 客户端能力，可向程序 B 发起结构化函数调用请求，支持多类型参数传入。

- 支持两种结果接收模式：

    - 同步模式：通过 HTTP 响应直接获取程序 B 的返回结果

    - 异步模式：建立 WebSocket 长连接，接收程序 B 主动推送的结果数据

- 具备参数组装、请求发送、结果解析的完整能力。

### 3\. 程序 B（执行端）功能

- 搭建 HTTP 服务端，监听并解析程序 A 的调用请求，根据请求中的函数标识匹配并执行对应业务函数。

- 支持两种结果回传方式：

    - 同步响应：在当前 HTTP 请求中直接返回执行结果与参数

    - 异步推送：通过已建立的 WebSocket 连接，主动向程序 A 推送执行结果 / 异步数据

- 内置函数映射注册机制，支持扩展新增可调用函数，无需修改通信核心代码。

### 4\. 参数传递规范

- 所有请求与回传数据采用结构化格式（推荐 JSON）封装，保证可读性与扩展性：

    - 请求报文：包含目标函数名称、入参参数集合

    - 响应报文：包含执行状态标识、返回值 / 结果数据集

- 支持数值、字符串、数组、对象等多类型参数的传输与解析。

## 三、技术实现要求

1. **核心开发**：业务逻辑、通信底层逻辑使用 C\+\+ 开发。

2. **界面与网络**：基于 Qt 框架实现图形界面，优先复用 Qt Network 模块完成 HTTP 服务 / 客户端、WebSocket 通信能力。

3. **协议合规**：严格遵循 HTTP 1\.1 标准与 WebSocket 标准协议。

## 四、Qt 界面需求

### 程序 A 界面

- 连接配置区：可配置程序 B 的 IP 地址、端口，选择通信模式（HTTP 同步 / WebSocket 异步）

- 函数调用区：下拉选择目标调用函数，可视化编辑输入参数，提供调用触发按钮

- 结果展示区：实时展示返回结果、调用状态与通信日志，支持异常信息高亮提示

### 程序 B 界面（可轻量化）

- 服务控制区：启动 / 停止 HTTP 与 WebSocket 服务，显示监听端口与服务运行状态

- 调用日志区：记录所有收到的调用请求、执行结果与异常信息

## 五、非功能需求

- 异常处理：覆盖连接失败、目标函数不存在、参数格式错误等场景，界面给出明确的错误提示

- 可扩展性：业务函数与通信层解耦，新增业务函数仅需注册函数映射，无需修改通信框架
